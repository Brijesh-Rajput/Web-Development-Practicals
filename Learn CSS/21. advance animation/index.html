Css allows animation of html elements without using js. u can chanage as many CSS properties as u want,as often as u wish. To use CSS animation,u must first specify some keyframes for the animation.
Keyframes hold what styles the element will have at certain times.
Properties are:- animation-name, animation-duration, animation-delay, animation-iteration-count, amimation-direction, animation-timing-function, animation-fill-mode.

animation-direction:-

It specifies whether an animation should be played forwards,backwards, or in alternate cycles. bydefault value is normal.
normal :- animation played as normals(forwards)
reverse :- animation played in reverse direction(backwards)
alternate :- animation is played forwards first, then backwards
alternate-reverse :- animation is played backwards first, then forwards.

animation start kaha se karni hai aur end kaha par karni hai.
Normal :- animation starts from 0% and goes to 100%
Reverse:- animation starts from 100% and goes to 0%

animation-fill-mode:-

It specifies style for target element when animation not playing.(before it starts,after it ends,or both).
none:- (default value) Animation will not apply any styles to element before or after it's executing.
forwards:- element will retain style values that is set by the last keyframe(depends on animation-direction and animation-iteration-count) How it depends on the animation-iteration-count ?? Answer:- let animation-direction:alternate; and forwards value for animation round odd and even is different.
backwards:- element will get style values that is set by the first keyframe (depends on animation-direction) , and retain this during the animation-delay period.
both:- animation will follow rules for both forwards and backwards,
